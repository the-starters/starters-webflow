wf-debug-css

Generate the debug embed for Webflow attribute migrations. Paste it into the page while you work in the Designer: elements still on the old grammar outline red, migrated ones green, and anything carrying both grammars flashes amber. Done means zero red, zero amber — then delete the embed and republish.

Red solid — still has old attributes
Green solid — migrated
Amber dashed — both grammars, remove the old one
Red banner — section root missing the wrapper attribute

Migration settings

CSS selector for the section being migrated — always page-specific, grab it from the live markup. Scoping keeps unrelated widgets (navbar Explore, other feeds) from being flagged.

Scope is required — an unscoped embed flags every matching element on the page.

Comma-separated. Leave out attributes that legitimately exist in both grammars (e.g. wf-xano-display) or they'll show as false reds.

The banner fires if the scope root is missing this attribute, written as name="value".

Selectors for empty/loader states that are display:none by default, so you can check them too. Comma-separated, scoped automatically.

Workflow

      
    
        
  1. Inventory the live markup first. Run snippet 2 on the published page — don't trust docs or memory. Build the per-element Remove → Add table from the dump.
    2. 
        
  2. Paste the debug embed into the page and publish (the Designer canvas shows outlines but is unreliable with the ⛔ labels — trust the published site).
    3. 
        
  3. Work the Designer until all green. Zero red, zero amber, no banner.
    4. 
        
  4. Verify from the browser. Snippet 2 returns [] and the new grammar renders — items present, states and pagination work.
    5. 
        
  5. Delete the embed, republish, and do a final functional check.
    6. 
      

    


    

      
Gotchas

      
    
        
  • Scope everything. Other widgets on the page may legitimately keep the old grammar — the navbar Explore mega-menu is wf-algolia and must stay.
    • 
        
  • Attributes shared by both grammars (like wf-xano-display on empty/loader elements) must not be in the old-attributes list.
    • 
        
  • Rendered clones can carry attributes copied from the template — inventory on a fresh load, and prefer checking the template element itself.
    • 
        
  • Webflow strips valueless custom attributes. Canonical markup is wf-xano-element="wrapper|list|template|empty|loader|error|…" (v0.4.0+); bare markers are legacy aliases.
    •
